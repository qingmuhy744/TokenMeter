**目标：** 实现 `TokenPlan` 模型的平滑迁移，支持父子结构，并将现有独立计划转化为“套餐”。

**架构：** 
1.  **0.3.0 (SQL)**: 添加 `parent_id` 和 `multiplier` 列。
2.  **0.3.1 (Func)**: 运行 `convert_to_suites` 函数，将所有顶层计划（无 parent_id）转化为套餐，并为每个套餐创建一个包含原模型配置的子计划。

**技术栈：** Python, SQLAlchemy, SQLite/PostgreSQL

```python
async def convert_to_suites(db):
    from sqlalchemy import select
    from backend.models import TokenPlan
    result = await db.execute(select(TokenPlan).where(TokenPlan.parent_id == None))
    plans = result.scalars().all()
    for p in plans:
        if not p.model: continue
        # 创建一个子项承接原有的模型配置
        child = TokenPlan(
            name=p.name,
            model=p.model,
            parent_id=p.id,
            multiplier=1.0,
            is_active=p.is_active,
            api_type=p.api_type,
            api_base=p.api_base,
            api_key=p.api_key,
            prompt=p.prompt,
            max_tokens=p.max_tokens,
            test_count=p.test_count,
            interval_minutes=p.interval_minutes
        )
        p.model = "" # 套餐本身不带模型名
        db.add(child)
    await db.commit()
```
