# Multi-Platform Docker CI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enable multi-platform Docker builds (AMD64 & ARM64) in GitHub Actions with optimized caching.

**Architecture:** Use `docker/setup-qemu-action` for ARM64 emulation, `docker/setup-buildx-action` for multi-platform build management, and `docker/build-push-action` with `type=gha` caching.

**Tech Stack:** GitHub Actions, Docker Buildx, QEMU, GHCR.

---

### Task 1: Update CI Workflow Configuration

**Files:**
- Modify: `.github/workflows/ci.yml`

- [ ] **Step 1: Add QEMU and Buildx setup steps**

Modify `.github/workflows/ci.yml` to insert setup steps before the Docker build.

```yaml
      - name: Set up QEMU
        uses: docker/setup-qemu-action@v3

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3
```

- [ ] **Step 2: Update Build and Push configuration**

Modify the `Build and push` step in `.github/workflows/ci.yml` to include `platforms`, `cache-from`, and `cache-to`.

```yaml
      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          platforms: linux/amd64,linux/arm64
          push: true
          tags: ${{ steps.tags.outputs.tags }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

- [ ] **Step 3: Verify workflow syntax**

Run a local lint check if possible, or visually verify the YAML structure. Ensure `platforms` and `cache-*` are correctly indented under `with`.

- [ ] **Step 4: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: enable multi-platform docker build with gha caching"
```
